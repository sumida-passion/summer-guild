# 夏休みギルド｜GPT引き継ぎガイド

この文書は、新しいChatGPTスレッドでも、現在の安定版・開発方針・納品手順を取り違えずに作業を続けるための固定引き継ぎ文書です。

`project-index.json` はリポジトリの構造やファイル情報を自動生成する「索引」です。設計思想、安定版の状態、GPTとの作業ルールは、この文書を正として扱います。

## 1. 新しいスレッドで最初に読む順番

1. `project-index.json`
2. `docs/GPT_HANDOFF.md`
3. `docs/development-log.md`
4. 必要に応じて `docs/game-rules.md` と対象ソースファイル

アップロードされたZIPがある場合は、そのZIPを最新の作業元として扱い、過去スレッドの推測や記憶で内容を補完しません。

- 算数ギルド：ギルドクエスト1〜10
- ギルドテスト：テスト1・2
- ギルドクエスト11以降：未実装・保留
- iPad Safari：動作確認済み

この状態を崩す変更は、ユーザーとの仕様確定前に実装しません。

## 3. 設計フェーズと実装フェーズ

ユーザーとの相談中は設計フェーズです。コード変更やZIP生成は行いません。

ユーザーが次の趣旨を明示した時点で、仕様を凍結し、実装フェーズへ移ります。

> 仕様はこれで確定。Zip作成作業に入って。

実装開始後に、未確定の追加提案を勝手に混ぜません。

## 4. 成果物を正とする原則

納品内容に関する説明は、予定、記憶、作業途中のフォルダから作りません。最後に生成した成果物そのものを確認した結果だけを報告します。

区別する状態：

- 読んだ
- 比較した
- 編集した
- ZIPへ収録した
- ZIPを再確認した

これらを同一視しません。「変更対象にした」ことは、「ZIPへ入った」ことの証明ではありません。

## 5. 差分ZIPの作成と確認

差分ZIPには、実際に変更または新規作成したファイルだけを、GitHubリポジトリと同じ相対パスで収録します。

納品前に必ず完成ZIPの内部一覧を直接取得し、期待する一覧と照合します。

- 期待一覧にあるがZIPにないファイル：0件
- ZIPにあるが期待一覧にないファイル：0件
- ZIP内ファイルを再展開し、作成した最終内容と一致すること

一致しない場合は納品しません。

## 6. `.github` などのGitHub管理領域

`.github/workflows/` 内のYMLは、GitHub Actionsの「仕組み」を定義するファイルです。画像やクエストを追加するたびに更新するものではありません。

現在の `generate-project-map.yml` は、リポジトリ情報を集めて `project-index.json` を自動生成する仕組みとして維持します。引き継ぎ文書を追加・更新するために、このYMLを変更する必要はありません。

将来 `.github` 配下の変更が本当に必要になった場合は、通常の差分ZIPとは分け、YMLファイル単体で渡し、ユーザーへ次の配置先を明示します。

```text
.github/workflows/
```

## 7. project-index.jsonの役割

`project-index.json` は自動生成される索引です。長期保存したい手書きのルールや設計判断を直接書き足しません。自動生成によって上書きされる可能性があるためです。

役割分担：

- `project-index.json`：ファイル構成・画像・JavaScript情報などの自動索引
- `docs/GPT_HANDOFF.md`：安定版・作業原則・スレッド引き継ぎ
- `docs/development-log.md`：実際に反映された変更履歴
- `docs/game-rules.md`：ゲーム仕様

## ふりかえりの修行（2026-07-23確定）

- デイリークエストは「百ます計算」または「ふりかえりの修行」から選ぶ。
- ふりかえりの修行は、教科→単元→基礎／標準／挑戦を自由選択する。ゲーム側は塾の進度を推測して「今日の復習」を指定しない。
- 初回2GPは教科・単元ごとではなく、全教科・全単元で共有する難易度別デイリーボーナス。基礎1回、標準1回、挑戦1回の最大6GP／日。
- 各セットは5問。成績GPは正確性ポイント×時間ポイント。再挑戦は可能で、初回2GP消化後も成績GPは獲得できる。
- 最初の実装単元は算数「単位量・平均」。国語・理科・社会は将来追加する。
- 単元一覧は塾の進捗管理ではなく、本人がその日に選ぶ復習教材の本棚として扱う。
- ペットは死なない・弱らない・餌を与えたときだけ喜ぶ加点型育成を構想中だが、現段階では実装保留。
