# GPT Zip作成・引き継ぎ標準手順

この文書は、新しいChatGPTスレッドでも安全に開発を継続するための必須運用ルールです。

## 標準フロー

1. ユーザーが最新版Zipをアップロードする。
2. GPTはZip内の `project-index.json` を最初に最後まで読む。
3. GPTはプロジェクト全体と今回の関連ファイルを確認する。
4. ユーザーとGPTは仕様検討を行う。この段階では完成Zipを作成しない。
5. ユーザーが「仕様はこれで確定だね、Zip作成作業に入って」または同等の明確な合図を出す。
6. GPTは元Zipを実際に展開し、対象ファイルを実際に編集する。
7. GPTは構文、ファイル構成、依存関係、保存データ互換性を検証する。
8. GPTは変更ファイル一覧を作る。
9. GPTは新しいZipを実際に生成する。
10. GPTは生成後のZipを再展開または一覧確認し、変更ファイルが入っていることを確認する。
11. 完成Zip、変更ファイル一覧、GitHubでの配置先、検証結果をユーザーへ渡す。

## 絶対禁止

- 実際に編集していないのに「完成しました」と言わない。
- 実際に生成していないZipへのリンクを完成版として提示しない。
- 元Zipの名前だけを変えて完成版にしない。
- 未確認の動作を「確認済み」と表現しない。
- 仕様確定の合図より前に実装を開始しない。

## 納品形式

原則は変更ファイルのみを同じフォルダ階層で収録した差分Zipとする。ユーザーが明示した場合のみ、プロジェクト丸ごとのZipを作成する。
